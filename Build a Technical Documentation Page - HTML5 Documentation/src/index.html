Introduction

HTML5 is the fifth and last major HTML version released in 22/01/2008 by the World Wide Web Consortium (W3C).

HTML5 is developed as a Living Standard, a concept that emphasizes on an incomplete but continually improving and updating project.

Improvments to HTML5 in particular include:

  • Scalable Vector Graphics (SVC)
  • Usage of Web SQL databases for temporary data storage
  • Usage of JavaScript within the web browser
  • Removal of outdated elements (tags)
  • New types of form control and elements.

This technical document details the new outlining and sectioning elements of HTML5, specific regarding <nav>, <section>, <article>, <header>, and <footer> elements.

nav tag

The HTML5 <nav> tag is also known as the navigation section element.

It represents a section of a page that provides navigation links. These navigation links can lead to either within or outside the current document.

The below is an example of how the current sidebar uses the <nav> tag:


<nav id="navbar">
  <header id="navheader">
    HTML5 Documentation
    </header>
    <a href="#introduction" class="nav-link">Introduction</a>
    <a href="#nav_tag" class="nav-link">nav tag</a>
</nav>
section tag

The HTML5 <section> tag is also known as the generic section element.

It represents a standalone section, without a more specific semantic element to represent it. <section> tags are commonly accompanied by a heading.

The following is this section's example:

<section class="main-section" id="section_tag">
      <header>section tag</header>
        <article>
          <p>
            The HTML5 <section> tag is also known as the generic section element.
          </p>
        </article>
</section>
article tag

The HTML5 <article> tag is also known as the article contents element.

It represents a self-contained composition, intended to be independently distributed or reusable. A single page, or even <section> tag, can hold multiple <article> elements.

To be specific, if it is more reasonable to be able to separately syndicate the contents of a section, by itself, independently, the <article> tag should be used, rather than the <section> tag

*In truth, if it were not for this project's specifications of needing a <section> per heading/topic, the more optimal design would have been to submit a single section from the main content, from Introduction to References, and cover each topic as articles. Of course, if the technical document was much larger, spanning several pages and, indeed, sections, then <section> tags would have been the most reasonable method of document outlining.

header tag

The HTML5 <header> tag represents introductory content.

It is not sectioning content. It does not introduce a new section to the outline.

It may contain heading elements (<h1> to <h6>), logos, search forms, author names, and other elements, but these are not required.

For a simple example:

<section class="main-section" id="article_tag">
  <header>article tag</header> 
</section>
footer tag

The HTML5 <footer> tag represents the end content of its nearest sectioning content or sectioning root element.

Information on the author of the section/article, copyright data, or links to related documents are generally contained in the tag. 

<footer id="copyright">
    Copyright © CopyrightYearHere  allemandi, All Rights Reserved
</footer>
